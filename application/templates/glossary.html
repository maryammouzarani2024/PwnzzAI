AI & LLM Security Glossary

This glossary provides brief definitions of key technical terms related to AI, large language models (LLMs), and their associated security issues discussed in this application.

Artificial Intelligence & Machine Learning

Artificial Intelligence (AI)

Computer systems that can perform tasks typically requiring human intelligence, such as understanding language, recognizing patterns, and making decisions.

Large Language Model (LLM)

AI models trained on vast amounts of text data to understand and generate human-like text. Examples include GPT, Claude, and Llama.

Machine Learning (ML)

A subset of AI where systems learn patterns from data without explicit programming. Related to Model Theft and Data Poisoning vulnerabilities.

Training Data

The dataset used to teach an AI model. Can be exploited through Data Poisoning attacks.

Model Weights

Numerical parameters learned during training that define a model's behavior. Target of Model Theft attacks.

RAG (Retrieval-Augmented Generation)

A technique that enhances LLM responses by retrieving relevant information from external sources before generating answers. Used in several demos including Misinformation and Sensitive Information Disclosure.

Prompt

The input text given to an LLM to generate a response. Can be manipulated through Direct Prompt Injection or Indirect Prompt Injection.

Hallucination

When an LLM generates false or misleading information presented as fact. Related to Misinformation vulnerabilities.

Sentiment Model

A machine learning model that analyzes text to determine emotional tone (positive, negative, or neutral). Used in this app to demonstrate Model Theft and Data Poisoning attacks.

Technical Concepts

API (Application Programming Interface)

A way for different software systems to communicate. LLM APIs allow applications to access AI models programmatically.

Tokens

Units of text that LLMs process (roughly 0.75 words per token). Important for understanding Unbounded Consumption attacks.

Fine-tuning

Additional training of a pre-trained model on specific data. Can be exploited through Data Poisoning.

Context Window

The amount of text an LLM can process at once. Related to Unbounded Consumption vulnerabilities.

Embeddings

Numerical representations of text used by LLMs. Often used in RAG systems for information retrieval.

Plugin / Function Calling

Allowing LLMs to execute external functions or tools. Can lead to Insecure Output Handling and Excessive Agency issues.

OWASP Top 10 for LLMs

A list of the most critical security risks for applications using Large Language Models, published by the Open Web Application Security Project.

Rate Limiting

Restricting the number of requests to prevent abuse. Essential mitigation for Unbounded Consumption attacks.
